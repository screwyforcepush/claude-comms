# v2.0.0 Release Notes — CC2 Retirement

**Date:** 2026-05-14
**Commit:** `9adb2929` — *Retire CC2: remove hooks, server, dashboard, scripts, and CC2 guides*
**Package:** `claude-comms` (npm) — version bumped from 1.0.46 to 2.0.0 at publish time
**Decision record:** [`cc2-retirement-survey.md`](cc2-retirement-survey.md)

## What changed

CC2 — the SQLite-backed Bun event hub (`apps/server/` on `:4000`), Vue 3 observability dashboard (`apps/client/` on `:5173`), Python hook scripts (`.claude/hooks/`), and lifecycle scripts (`scripts/`) — was fully retired. The entire CC2 surface had been functionally replaced by:

- **Inter-agent messaging:** `.agents/tools/workflow/agent-comms.mjs` (Convex-backed, cursor-tracked, namespace-aware)
- **Orchestration and observability:** the Convex workflow engine (`workflow-engine/` + `convex/`) and its UI on `:3500`

### Deleted

| Surface | Path | Replacement |
|---------|------|-------------|
| Hook scripts (9 Python files) | `.claude/hooks/` | `agent-comms.mjs` + workflow engine job records |
| Hook wiring | `.claude/settings.json` | Not needed — no hook-based integration |
| Event hub server | `apps/server/` | Convex backend |
| Observability dashboard | `apps/client/` | Workflow engine UI (`:3500`) |
| Lifecycle scripts | `scripts/` | Workflow engine runner daemon |

## What v2.0.0 means for consumers

| Before (1.x) | After (2.0.0) |
|---------------|---------------|
| `npx claude-comms` installed `.claude/hooks/`, `.claude/settings.json`, and `.agents/` | `npx claude-comms` installs only `.agents/` and `CLAUDE.md` |
| Clients needed a running CC2 server for inter-agent messaging and observability | Clients connect to the Convex workflow engine — no local server required |
| Python hooks captured every harness event to SQLite | Job-level metrics flow through the workflow engine natively |

Clients that depend on CC2 hooks or the observability server should pin to `claude-comms@1.x`.

## Publish flow (user-driven)

The retirement commit includes `[skip ci]` in the body to prevent auto-publish on push. The v2.0.0 publish is a deliberate user action:

1. **Push** the retirement commit to `main`
2. **Trigger** `workflow_dispatch` on the *Publish Claude Comms Setup Installer* workflow with `version_bump: major`
3. The workflow runs: lint, build, `npm version major` (1.0.46 -> 2.0.0), then `npm publish --access public`
4. **Auth:** npm trusted-publisher OIDC — the GitHub Actions runner gets a short-lived token automatically. No `NPM_TOKEN` secret needed.
5. The workflow commits the version bump and pushes a `v2.0.0` tag

If OIDC fails, manual fallback: `NPM_TOKEN` from `.env` (gitignored, not committed). Token has a ~2-day expiry from 2026-05-14.

## Known gaps

1. **`packages/setup-installer/src/`** — installer source code (`fetcher/github.js`, `utils/constants.js`, `cli/runner.js`) still references `.claude/hooks/comms/*.py` paths that no longer exist in the repo. Whether the v2.0.0 install gracefully skips missing files or hard-fails depends on the fetcher's error handling. Flagged for verification before or after publish.

2. **`packages/setup-installer/test/`** — test fixtures and mocks reference CC2 endpoints (`localhost:4000`), hook paths, and `Multi-Agent Observability` strings. These are implementation-code tests (out of scope for the retirement commit) but will need updating to match the v2.0.0 installer behavior.

3. **`docs/project/guides/consultant-paradigm-guide.md`** — marked as superseded. The proxy pattern it describes is replaced by the workflow engine's direct harness execution. The `agent_job.py` adapter still exists for ad-hoc use but the `--consultant` template injection no longer functions.

4. **`docs/project/guides/orchestration-guide.md`** — describes CC2-era batch orchestration patterns (agent naming, broadcast/inbox messaging). The patterns are conceptually valid but the communication layer they reference (CC2 hooks) is gone. The guide doesn't directly name CC2 but its messaging patterns assume the old `send_message.py`/`get_unread_messages.py` plumbing. May warrant a refresh to reference `agent-comms.mjs` instead.

5. **`.github/workflows/publish-claude-comms.yml` line 250** — failure notification still mentions "NPM_TOKEN not configured" as a common issue. With OIDC, this is stale guidance. Minor cosmetic fix.
